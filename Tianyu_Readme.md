### 无人机藻类识别项目 - AI视觉（Tianyu）专属 README
#### 一、核心定位
作为项目AI视觉算法负责人，聚焦藻类视觉识别全流程，基于贺一冉交付的标准化预处理图像，完成藻类像素级定位、分类、量化，输出标准化结果至数模组，不参与数据采集、预处理等非视觉算法环节。

#### 二、绝对清晰的工作边界
✅ 核心负责（AI视觉全流程）
1. 基于贺一冉输出的预处理干净图像，开展数据集标注复核
2. 搭建、训练、调优U-Net藻类语义分割模型（像素级定位+多类别区分）
3. 执行模型推理：识别藻类有无、位置、门类（蓝藻/绿藻/其他）
4. 量化计算：藻类覆盖面积、占比、浓度等级（高/中/低）
5. 输出标准化识别结果，对接数模组

❌ 绝不涉及（归数据/航拍/数模模块）
1. 不参与任何原始图像采集、预处理（去雾/去反光/色彩校正等）
2. 不使用LabelMe开展初始像素级标注（仅复核）
3. 不进行时序分析、预测建模等数模环节工作
→ 以上全部归：贺一冉 + 航拍组 + 数模组

#### 三、分步执行任务（按顺序推进）
1. 数据接收与核验
   - 接收贺一冉交付的：预处理干净图像集、LabelMe标注文件（JSON）、掩码图
   - 核验数据完整性、标注准确性，反馈异常至贺一冉修正
   - 输出：核验通过的标准化训练数据集

2. 模型搭建与训练
   - 基于U-Net架构搭建语义分割模型，适配标注数据格式
   - 划分训练/验证/测试集，调优超参数（学习率、批次大小等）
   - 训练目标：精准区分藻类（蓝/绿/其他）、清水、泥沙、阴影
   - 输出：训练完成且验证达标的藻类分割模型

3. 模型推理与量化计算
   - 输入：单张预处理干净图像
   - 输出：
     - 藻类像素级定位掩码图（区分门类）
     - 藻类总面积、覆盖率（占图像比例）
     - 各类型藻占比、浓度等级（高/中/低）

4. 结果交付与对接
   - 按标准接口整理识别结果（数值+可视化图）
   - 交付至数模组，不参与后续分析/预测环节
   - 输出：标准化交付结果包

#### 四、模块分工（一句话界定）
- 航拍组：采集原始无人机图像
- 贺一冉：原始图像→预处理干净图像→LabelMe标注数据集
- 本人：标注数据集→U-Net模型训练→藻类识别/量化结果
- 数模组：识别量化结果→时序分析/预测建模

#### 五、最终交付物
1. 训练调优完成的U-Net藻类语义分割模型文件（含权重）
2. 模型推理代码（可独立运行，输出量化结果）
3. 藻类识别可视化结果图（掩码叠加原图）+ 面积/占比/浓度数值表
4. 数模组对接的标准输出接口文档（数据格式/传输规范）
