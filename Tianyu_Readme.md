无人机藻类识别项目 - 本人（AI视觉）专属 README
 
（严格按分工：贺一冉负责预处理，我只做AI视觉模型与识别推理）
 
一、我的核心定位
 
我是项目AI视觉算法负责人，只负责：在贺一冉处理好的干净图像上，做藻类定位、分类、量化，并输出标准结果给数模组。
 
二、绝对清晰的工作边界
 
✅ 我只做这些
 
1. 用贺一冉输出的已预处理完的干净图像做数据集标注
2. 搭建、训练藻类语义分割模型（像素级定位+分类）
3. 模型推理：识别有无藻、藻在哪、是哪类藻
4. 计算藻类覆盖面积、占比、浓度等级
5. 将结果交给数模模块
 
❌ 我完全不做
 
1. 不做任何图像预处理（去雾、去反光、色彩校正等）
2. 不飞无人机、不拍图
3. 不做数据采集、数据清洗
4. 不做时序分析、预测建模
→ 以上全部归：贺一冉 + 航拍组 + 数模组
 
三、我具体要干的事（按顺序）
 
1. 接收贺一冉的输出
拿到：已去反光、去雾、增强后的标准无人机图像
2. 数据集标注
对干净图像做像素级标签：
- 藻类 / 非藻类
- 蓝藻 / 绿藻 / 其他藻
3. 模型搭建与训练
使用语义分割模型（U-Net）实现：
- 藻类像素级定位
- 藻类门类区分
- 抗干扰（区分水草、泥沙、阴影）
4. 模型推理与量化计算
输入：贺一冉预处理好的单张图
输出：
- 藻类定位掩码图
- 藻类总面积、覆盖率
- 各类型藻占比
- 浓度等级（高/中/低）
5. 结果交付
把识别结果传给数模模块，不参与后续预测。
 
四、模块分工（一句话分清）
 
- 航拍组：拍原图
- 贺一冉：原图 → 预处理干净图
- 我：干净图 → 藻类定位/分类/量化结果
- 数模组：量化结果 → 分析/预测
 
五、我的交付物
 
1. 训练好的藻类分割模型
2. 模型推理代码
3. 藻类识别结果图 + 面积/浓度数据
4. 给数模组的标准输出接口
