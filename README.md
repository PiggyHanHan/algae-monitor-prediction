1. 项目概述
 
本项目针对湖泊/水库藻华爆发监测需求，搭建**「无人机数据采集 → 图像预处理与标注 → AI视觉分割 → 数模时序预测 → 交互式可视化展示」**全链路系统。核心解决传统藻类监测成本高、频次低、数据滞后问题，实现藻类精准识别、时空分布量化与未来趋势预警。
 
大创阶段以完成技术闭环、本地可演示Demo为核心目标，不追求公网部署、每日高频采集等过度工程化内容；长期拓展为可落地的水体监测平台，支撑环保部门决策。
 
2. 项目核心目标
 
2.1 大创阶段核心目标（1年内验收标准）
 
1. 数据层：完成高质量无人机藻类数据集构建（含原始采集、预处理、像素级标注），数据无错标、漏标、格式混乱。
2. 算法层：基于U-Net实现藻类像素级语义分割，完成有无藻识别、藻的精准定位、蓝藻/绿藻/其他藻分类、覆盖面积/浓度等级量化四大核心功能，模型分割精度满足演示需求。
3. 数模层：基于AI视觉输出的时序实测数据，搭建藻类增长预测模型，输出未来1-2个月藻类分布、面积变化与风险等级，预测数据格式与实测数据完全对齐。
4. 展示层：开发本地可运行交互式可视化Demo，实现三大核心功能：① 拖动时间进度条切换不同日期；② 一键切换“实测模式/预测模式”查看藻类分布图；③ 同步展示面积、覆盖率、浓度等级等量化数据；演示采用「少量真实实测数据+模拟预测数据」，无需每日拍摄更新。
5. 协同层：各模块数据对接顺畅，无跨模块越界工作，全流程可现场操作演示，满足结题答辩验收要求。
 
2.2 长期拓展目标（1年后进阶方向）
 
1. 数据层：支持周期性（每周/每月）无人机数据更新，实现模型自动迭代优化。
2. 算法层：优化分割模型鲁棒性，适配水草、阴影、强光反光等复杂水体环境；拓展藻类细分物种识别（可选）。
3. 展示层：新增“新图片一键识别预测”功能，支持用户上传新航拍图获取结果；可选局域网部署，满足定点监测使用需求。
4. 应用层：对接环保部门数据平台，实现监测数据同步上报，支撑实际水质管理工作。
 
3. 技术模块与分工边界
 
项目分为四大核心模块，分工严格隔离、数据单向流转、无交叉重叠工作，各模块仅负责自身职责，通过标准化文件完成对接。
 
3.1 模块划分与核心职责
 
3.1.1 数据采集与预处理标注模块（负责人：贺一冉）
 
核心定位：项目数据源头，负责从原始航拍图到可用数据集的全流程处理，为后续模块提供唯一、高质量输入。
具体职责：
 
1. 数据采集配合：对接航拍组，明确拍摄规范（垂直90°俯拍、M档RAW、10-50m高度、无强光反光），收集整理原始航拍图像，按日期/区域归档，确保数据完整无缺失。
2. 图像预处理：对原始图像执行去反光、去雾、色彩校正、畸变矫正、尺寸统一，输出干净、无干扰、藻类特征清晰的标准图像；不参与任何模型训练与标注之外的工作。
3. 像素级标注：使用LabelMe工具，对预处理后的干净图像完成标注，标注类别严格统一：蓝藻、绿藻、其他藻类、清水、泥沙、阴影；确保标注边界精准、无错标/漏标，生成JSON标注文件与像素掩码图。
4. 数据标准化：整理输出完整数据集，包含：预处理干净原图、LabelMe JSON标注文件、像素掩码图；撰写数据规范文档（含拍摄要求、预处理参数、标注规则），确保后续模块可直接使用。
不涉及工作：不参与深度学习模型训练、藻类量化计算、数模预测、前端可视化开发、展示端对接等所有后续模块任务。
 
3.1.2 AI视觉语义分割与量化模块（负责人：吴天宇、刘俊辉）
 
核心定位：项目算法核心，负责从标注数据集到藻类识别、量化结果的全流程输出，为数模、展示模块提供核心数据支撑。
具体职责：
 
1. 模型搭建与训练：接收贺一冉交付的标准化数据集，基于U-Net搭建藻类语义分割模型，完成模型搭建、训练、验证、优化；确保模型能精准区分藻类与清水、泥沙、阴影等干扰项。
2. 推理与量化计算：基于训练好的模型完成推理，输出藻类分割掩码图；基于掩码图计算藻类总覆盖面积、各门类藻类占比、整体覆盖率、浓度等级（高/中/低），生成结构化量化数据（JSON/CSV格式）。
3. 数据格式对齐：接收数模组的预测数据，将实测数据与预测数据统一格式化，严格对齐时间格式（YYYY-MM-DD）、类别标签、图像尺寸，确保展示端可正常加载。
4. 展示端素材对接：整理分割掩码图、量化数据，作为展示端的核心原材料，配合乔梓阁完成数据对接验证，不参与前端代码开发。
不涉及工作：不参与图像预处理、LabelMe标注、数模模型构建、前端可视化开发、展示端部署等非算法模块任务。
 
3.1.3 数模时序预测模块（负责人：数模组）
 
核心定位：项目趋势预警核心，基于AI视觉的时序实测数据，完成藻类未来分布与风险预测。
具体职责：
 
1. 预测模型搭建：接收吴天宇、刘俊辉交付的藻类时序实测数据（含日期、面积、浓度、类别占比），搭建藻类增长时序预测模型，完成模型训练与调优。
2. 趋势预测与风险评估：基于训练好的模型，推演未来1-2个月藻类时空分布、覆盖面积变化，输出藻华爆发风险等级，生成标准化预测数据（格式与实测数据完全一致）。
3. 数据交付：将预测数据按规范整理，交付给吴天宇、刘俊辉完成格式对齐，不参与图像识别、前端开发工作。
不涉及工作：不参与无人机图像预处理、标注、U-Net模型训练、前端可视化、展示端开发等非数模任务。
 
3.1.4 交互式可视化展示模块（负责人：乔梓阁）
 
核心定位：项目成果展示端，负责将算法、数模的核心结果转化为可交互的本地Demo，支撑结题演示。
具体职责：
 
1. 展示端开发：基于Python（Streamlit/Dash优先）搭建前后端框架，实现本地可运行的可视化程序，不涉及公网部署与服务器运维。
2. 核心交互功能实现：
- 搭建时间进度条，支持拖动选择任意日期，加载对应实测/预测数据；
- 实现“实测模式/预测模式”一键切换，动态渲染藻类分布图像；
- 同步展示量化数据（面积、覆盖率、浓度等级、类别占比），确保数据加载正常、展示清晰无重叠。
3. 接口对接：接收吴天宇、刘俊辉交付的实测/预测图像与结构化数据，搭建后端数据接口，实现数据正常读取与渲染，不参与数据预处理、模型训练。
4. 演示优化：打磨操作流程，确保现场演示流畅，支持快速切换不同日期、查看对比结果，满足结题答辩演示要求。
不涉及工作：不参与数据采集、预处理、标注、AI模型训练、数模预测计算等所有前端之外的模块任务。
 
4. 项目全流程执行链路（分步执行，不可跳步）
 
严格按照以下9步推进，前一模块完成全部工作并确认无误后，再将数据交付下一模块，避免流程混乱：
 
1. 航拍组按规范完成水体无人机航拍，输出原始航拍图像，交付给贺一冉。
2. 贺一冉对原始图像完成预处理，输出干净标准图像，交付给吴天宇、刘俊辉与自身用于标注。
3. 贺一冉使用LabelMe完成干净图像的像素级标注，生成完整标注数据集，交付给吴天宇、刘俊辉。
4. 吴天宇、刘俊辉基于标注数据集，完成U-Net模型训练与推理，输出藻类分割掩码图与实测量化数据，交付给数模组与自身用于格式对齐。
5. 数模组接收实测时序数据，完成预测模型搭建与未来藻类分布推演，输出标准化预测数据，交付给吴天宇、刘俊辉。
6. 吴天宇、刘俊辉将实测数据与预测数据完成格式统一，整理为展示端所需的图像与结构化数据，交付给乔梓阁。
7. 乔梓阁接收素材，完成展示端前后端开发与交互功能实现，搭建本地可运行Demo。
8. 全团队开展联调测试：验证各模块数据对接、展示端功能完整性，修复加载异常、展示混乱等问题。
9. 完善文档、打磨演示流程，准备结题答辩与验收。
 
5. 各模块核心交付物
 
5.1 数据采集与预处理标注模块（贺一冉）

5.4 交互式可视化展示模块（乔梓阁）

6.1 第一阶段：数据准备期（第1-2个月）
 
核心任务：完成数据集构建，为算法开发打基础。
工作内容：航拍组完成图像采集；贺一冉完成预处理、全量LabelMe标注；团队验收数据集，确认无错标、漏标，符合模型训练要求。
 
6.2 第二阶段：算法模型开发期（第3-5个月）
 
核心任务：完成AI视觉与数模模型搭建、训练、优化，实现核心功能。
工作内容：吴天宇、刘俊辉完成U-Net模型训练、推理、量化计算；数模组完成预测模型搭建、趋势推演；两组完成数据格式对齐与对接验证。
 
6.3 第三阶段：展示端开发与集成期（第6-8个月）

核心任务：完善成果、文档，完成答辩验收。

7. 重要约定（核心执行规则）
 
7.1 数据格式统一约定
 
1. 时间格式：所有数据统一采用 YYYY-MM-DD ，无其他格式变体；
2. 类别标签：统一为 blue_algae （蓝藻）、 green_algae （绿藻）、 other_algae （其他藻类）、 clear_water （清水）、 sediment （泥沙）、 shadow （阴影），各模块不得擅自修改；
3. 图像尺寸：统一采用 1024×1024 （或其他固定统一尺寸），确保跨模块加载无适配问题；
4. 文件格式：结构化数据采用JSON/CSV，图像采用PNG格式，标注文件采用LabelMe标准JSON。
 
7.2 分工边界约定
 
1. 各模块仅负责自身职责，不得擅自承接其他模块任务（如贺一冉不碰模型、乔梓阁不碰标注）；
2. 数据交付需附带交付说明文档，明确数据内容、格式、适用模块，避免对接误解；
3. 跨模块问题需通过沟通解决，不得擅自修改其他模块交付数据，如需修改需经原模块负责人同意。
 
7.3 大创验收优先级约定
 
1. 优先完成技术闭环与本地Demo演示，不追求每日无人机拍摄、公网部署、长期运营等超出学生项目范围的内容；
2. 展示端演示采用「3-5组真实实测数据+模拟预测数据」，重点展示“数据输入→模型处理→结果展示”的完整流程，而非实际运营监测；
3. 答辩演示优先保障操作流畅性、功能完整性，其次追求数据细节与模型精度。
 
7.4 进度与沟通约定
 
1. 每周开展一次团队同步会，各模块汇报进度、反馈问题、协调对接；
2. 核心节点（数据集交付、模型训练完成、展示端Demo上线）需提前1天通知对接模块，预留验收时间；
3. 出现跨模块问题，需在24小时内沟通解决，避免延误项目整体进度。
