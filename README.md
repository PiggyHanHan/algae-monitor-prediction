📋 项目总览

1.1 项目背景

针对校园池塘、城市湖泊水体藻类爆发监测痛点，基于无人机航拍图像，通过计算机视觉（CV）深度学习实现藻类区域识别、种类分类、覆盖率计算，结合数学建模完成藻类浓度/扩散趋势预测，最终通过Web可视化平台展示结果，为水体环保监测提供轻量、高效、低成本的AI解决方案。

1.2 核心定位

- 赛道：AI+环保+遥感视觉

- 技术栈：PyTorch深度学习 + 时序/统计建模 + Web前端可视化

- 落地形式：PC端算法模型 + Web演示平台（不部署嵌入式、不开发无人机硬件）

- 目标：大创省级/国家级立项，打造大一CV标杆项目

1.3 核心优势

1. 硬件现成：依托图像识别实验室无人机，零设备成本

2. 场地就近：东九教学楼池塘、喻家湖实景采集

3. 团队闭环：算法+建模+数据+文档+前端完整配置

4. 技术适配：纯AI视觉方向，贴合实验室主业，无冗余技术

5. 可行性拉满：大一可实现，无硬核门槛，成果可展示

🔗 项目全链路技术方案

4.1 整体流程（端到端·无嵌入式）

无人机实景采集 → 数据清洗/预处理 → 标注构建数据集 → CV深度学习藻类识别 → 数学建模藻类预测 → Web可视化展示 → 成果输出

4.2 模块1：数据层（贺一冉 主负责）

4.2.1 数据来源

1. 实景采集：实验室无人机拍摄东九池塘、喻家湖航拍图

2. 公开数据集：水体藻类遥感公开数据集（补充训练）

4.2.2 数据处理

1. 清洗：剔除模糊、过曝、无效图像

2. 预处理：尺寸统一、去雾、归一化、数据增强

3. 标注：使用LabelMe标注藻类区域（分割/分类）

4. 输出：标准化数据集（训练集/验证集/测试集）

4.2.3 数据规模（申报前）：50-100张实景图（结题：500-1000张）

4.3 模块2：CV深度学习藻类识别（牵头人/刘俊辉 主负责）

4.3.1 核心任务

1. 二分类：藻类/非藻类判别

2. 语义分割：藻类区域提取、覆盖率计算

3. 精度目标：申报前≥70%，结题≥85%

4.3.2 技术选型

- 框架：PyTorch

- 模型：轻量CNN、U-Net（适合小样本、航拍场景）

- 流程：数据集加载 → 模型搭建 → 训练 → 评估 → 模型导出

4.3.3 输出：藻类位置、覆盖率、分类结果

4.4 模块3：数学建模藻类预测（刘俊辉/牵头人 主负责）

4.4.1 核心任务

1. 藻类浓度预测

2. 藻类扩散趋势分析

4.4.2 技术选型

1. 基础模型：线性回归、随机森林、XGBoost

2. 进阶模型：LSTM时序预测（适配时间序列数据）

3. 特征：覆盖率、水质参数、温度、光照

4.4.3 输出：预测曲线、扩散热力图、结论分析

4.5 模块4：Web可视化展示（发小 主负责）

4.5.1 核心功能

1. 图片上传：无人机航拍图输入

2. 算法调用：识别+预测结果展示

3. 可视化：藻类标记图、覆盖率、预测曲线

4. 项目介绍：背景、团队、技术、成果

4.5.2 技术栈：前端（JS/Vue）+ 后端（Go）

4.5.3 定位：纯演示平台，不部署模型，仅展示结果

4.6 模块5：文档与成果（贺一冉 主负责）

1. 大创申报书、中期报告、结题报告

2. 项目论文、实验数据、图表

3. 答辩PPT、演示视频、成果手册
