# 📋 无人机航拍藻类智能识别与藻华预警系统 —— 团队主README

*（根据最终方案修正，纯文字结构化，适配大创项目文档/汇报直接使用）*

## 🎯 1. 项目概述

本项目针对湖泊/水库藻华爆发监测的现实需求，搭建 **「无人机数据采集 → 图像预处理与标注 → AI视觉分割 → 数模时序预测 → 交互式可视化展示」** 全链路系统。核心解决传统藻类监测成本高、频次低、数据滞后、精准度不足四大痛点，实现藻类精准识别、时空分布量化、未来趋势预警与成果直观呈现。

大创阶段以完成技术闭环、落地本地可演示Demo为核心验收目标，不追求公网部署、每日高频采集等过度工程化内容；长期可拓展为可落地的水体监测平台，支撑环保部门水质安全管理与决策。

## 🎯 2. 项目核心目标

### 2.1 大创阶段核心目标（1年内验收标准）

1. **数据层**：完成高质量无人机藻类数据集构建，含原始航拍采集、专业图像预处理、像素级LabelMe标注，数据无错标、漏标、格式混乱，可直接用于深度学习模型训练。
2. **算法层**：基于U-Net实现藻类像素级语义分割，达成四大核心功能：① 精准识别有无藻类；② 定位藻类空间分布；③ 区分蓝藻/绿藻/其他藻类门类；④ 量化藻类覆盖面积（绝对面积，平方米）。
3. **数模层**：基于AI视觉输出的时序实测数据（藻类面积等），搭建藻类增长预测模型，输出未来1-2个月藻类分布、面积变化与藻华风险等级，预测数据格式与实测数据完全一致。
4. **展示层**：开发本地可运行交互式可视化Demo，实现核心交互：① 拖动时间进度条切换不同日期；② 一键切换“实测模式/预测模式”查看藻类分布图；③ 同步展示面积等量化数据；演示采用「3-5组真实实测数据+模拟预测数据」，无需每日拍摄更新。
5. **协同层**：各模块数据对接顺畅、分工边界清晰，全流程可现场操作演示，满足结题答辩验收要求。

### 2.2 长期拓展目标（1年后进阶方向）

1. **数据层**：支持周期性（每周/每月）无人机数据更新，实现模型自动迭代优化，提升识别与预测精准度。
2. **算法层**：优化模型鲁棒性，适配水草、阴影、强光反光等复杂水体环境；拓展藻类细分物种识别（可选）。
3. **展示层**：新增“新图片一键识别预测”功能，支持用户上传新航拍图获取结果；支持局域网部署，满足定点监测使用需求。
4. **应用层**：对接环保部门数据平台，实现监测数据同步上报，支撑实际水质管理工作。

## 👥 3. 技术模块与分工边界

项目分为四大核心模块，分工严格隔离、数据单向流转、无交叉重叠工作，各模块仅负责自身职责，通过标准化文件完成对接。

### 🗂️ 3.1 数据采集与预处理标注模块（负责人：贺一冉）

**核心定位**：项目数据源头，负责从原始航拍图到标准化数据集的全流程输出，为后续模块提供唯一、高质量输入。

**具体职责**（✅ 已明确）
- **数据采集配合**：对接航拍组，明确拍摄规范（垂直90°俯拍、M档RAW、10-50m高度、无强光反光），整理原始航拍图像，按日期/区域归档，确保数据完整无缺失。
- **图像预处理**：使用 AI 视觉模块提供的统一预处理脚本，对原始图像执行：
  - 画质修复：去反光、去雾、色彩校正、畸变矫正
  - 几何校正：根据无人机高度、相机参数及姿态角（俯仰、滚转、偏航）将图像校正为正射视角，**统一地面分辨率**（例如 0.02 米/像素），并缩放到固定尺寸（例如 2048×2048）
- **像素级标注**：使用LabelMe工具，对**预处理后的标准图像**完成标注，标注类别统一：蓝藻、绿藻、其他藻类、清水、泥沙、阴影，生成JSON标注文件与像素掩码图。
- **数据标准化**：整理完整数据集（含预处理原图、JSON标注、像素掩码图），撰写数据规范文档（拍摄要求、预处理参数、标注规则）。
- **元数据台账**：提供CSV格式元数据台账，包含每张图像的拍摄时间、高度、姿态角、GPS、相机参数等（详见3.5元数据规范）。

**不涉及工作**：不参与深度学习模型训练、藻类量化计算、数模预测、前端可视化开发、展示端对接。

### 🤖 3.2 AI视觉语义分割与量化模块（负责人：吴天宇、刘俊辉）

**核心定位**：项目算法核心，负责从标注数据集到藻类识别、量化结果的全流程输出，为数模、展示模块提供核心数据支撑。

**具体职责**（✅ 已明确）
- **统一预处理脚本开发**：编写并提供集成画质修复+几何校正的预处理脚本，供数据端使用；负责脚本的维护、bug修复与版本升级。
- **模型搭建与训练**：接收贺一冉交付的数据集，基于U-Net搭建藻类语义分割模型，完成模型搭建、训练、验证、优化，确保精准区分藻类与清水、泥沙、阴影等干扰项。
- **推理与量化计算**：基于训练好的模型推理，输出藻类分割掩码图；**直接计算藻类覆盖面积（平方米）**（基于统一地面分辨率，无需额外换算），以及各门类面积，生成JSON格式结构化量化数据。
- **数据格式对齐**：接收数模组预测数据，统一实测/预测数据格式（时间、类别、图像尺寸），适配展示端加载需求。
- **展示端素材对接**：整理分割掩码图、量化数据，作为展示端核心原材料，配合完成数据对接验证。

**不涉及工作**：不参与图像预处理、LabelMe标注、数模模型构建、前端可视化开发、展示端部署。

- **趋势预测与风险评估**：推演未来1-2个月藻类面积变化、藻华爆发风险等级，生成标准化预测数据（格式与实测数据完全一致）。
- **数据交付**：将预测数据整理交付，由吴天宇、刘俊辉完成格式对齐。

### 📋 3.5 数据格式统一约定（✅ 已明确）

- **时间格式**：统一采用 `YYYY-MM-DD HH:MM:SS` ，无其他变体；
- **类别标签**：统一为 `blue_algae`（蓝藻）、`green_algae`（绿藻）、`other_algae`（其他藻）、`clear_water`（清水）、`sediment`（泥沙）、`shadow`（阴影），不得擅自修改；
- **图像尺寸**：统一为固定尺寸（如 2048×2048），**统一地面分辨率**（如 0.02 米/像素），确保跨模块空间尺度一致；
- **文件格式**：结构化数据用JSON/CSV，图像用PNG，标注文件用LabelMe标准JSON；
- **元数据台账（CSV）**：必须包含以下字段（与AI模块JSON输出中的`drone_metadata`对齐）：
  | 字段名 | 类型 | 说明 |
  |--------|------|------|
  | `image_id` | string | 原始图像文件名（不含扩展名） |
  | `timestamp` | string | 拍摄时间，格式`YYYY-MM-DD HH:MM:SS` |
  | `height_m` | float | 相对水面高度（米） |
  | `pitch_deg` | float | 俯仰角（度） |
  | `roll_deg` | float | 滚转角（度） |
  | `yaw_deg` | float | 偏航角（度） |
  | `gps_lat` | float | GPS纬度 |
  | `gps_lon` | float | GPS经度 |
  | `camera_focal_mm` | float | 镜头焦距（毫米） |
  | `sensor_width_mm` | float | 传感器宽度（毫米） |
  | `sensor_height_mm` | float | 传感器高度（毫米） |
- **分割掩码图规范**：PNG格式，命名`{image_id}_seg.png`，颜色映射固定为：
  - 蓝藻：RGB(0,0,255)
  - 绿藻：RGB(0,255,0)
  - 其他藻类：RGB(255,255,0)
  - 清水：RGB(0,0,0)
  - 泥沙：RGB(128,128,128)
  - 阴影：RGB(64,64,64)

## 🔗 4. 项目全流程执行链路（不可跳步）

严格按以下9步推进，前一模块完成全部工作并确认无误后，再交付下一模块，避免流程混乱：

1. 航拍组按规范完成水体无人机航拍，交付原始航拍图像给贺一冉。
2. 贺一冉对原始图像完成预处理，输出干净标准图像，交付吴天宇、刘俊辉（用于模型训练）与自身（用于标注）。
3. 贺一冉使用LabelMe完成干净图像像素级标注，生成完整标注数据集，交付吴天宇、刘俊辉。
4. 吴天宇、刘俊辉基于标注数据集，完成U-Net模型训练与推理，输出分割掩码图、实测量化数据（绝对面积等），交付数模组（用于预测）与自身（用于格式对齐）。
5. 数模组接收实测时序数据，完成预测模型搭建与未来藻类面积变化推演，输出标准化预测数据，交付吴天宇、刘俊辉。
6. 吴天宇、刘俊辉完成实测+预测数据格式统一，整理为展示端素材，交付乔梓阁。
7. 乔梓阁接收素材，完成展示端前后端开发与交互功能搭建，搭建本地可运行Demo。
8. 全团队联调测试：验证模块数据对接、展示端功能完整性，修复加载异常、展示问题。
9. 完善文档、打磨演示流程，准备结题答辩与验收。
