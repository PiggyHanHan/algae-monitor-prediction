无人机藻类识别项目 - 数据链路（贺一冉）专属README
 
一、核心工作目标
 
全权负责项目全链路数据筹备，从原始航拍图像采集、清洗优化到像素级标注，最终输出干净、标准化、可直接用于U-Net训练的藻类数据集，为AI视觉模型端（你+刘俊辉）提供唯一、高质量的数据输入。
 
二、严格职责边界（不混淆、不越界）
 
✅ 必须完成（数据端全流程）
 
1. 配合航拍组完成无人机图像采集，严格遵循拍摄规范
2. 对原始图像做全流程图像预处理，输出无干扰、高清晰度的干净图像
3. 使用LabelMe工具完成像素级数据标注，划分藻类类别与非藻类区域，生成标准标注文件
 
❌ 绝不涉及（完全归你+刘俊辉）
 
1. 不做U-Net模型搭建、训练与推理
2. 不计算藻类面积、覆盖率、浓度等级
3. 不参与模型结果对接、量化分析与预测建模
→ 以上全部归：AI视觉模块（你+刘俊辉）
 
三、分步执行任务（按顺序执行，不跳步）
 
第1步：数据采集配合（源头把控）
 
1. 对接航拍组，明确拍摄要求：垂直90°俯拍、M手动档、RAW格式、10-50m高度、无强光反光
2. 收集并整理所有原始航拍图像，按拍摄时间/区域分类归档，避免混乱
3. 输出：原始无人机航拍图像集（无缺失、无格式错误）
 
第2步：图像预处理（核心修图工作）
 
对原始图像做全流程清洗优化，确保图像干净、藻类特征清晰：
 
1. 去除水面高光反光、雾感/噪点
2. 色彩校正、图像增强（提升藻类与清水/泥沙对比度）
3. 畸变矫正、图像尺寸统一（适配后续标注与模型输入）
4. 输出：预处理干净图像集（无反光、无杂色、藻类边界清晰）
 
第3步：LabelMe像素级标注（数据标准化）
 
使用LabelMe工具，对干净图像完成精准标注，为U-Net提供“标准答案”：
 
1. 标注类别统一：藻类（蓝藻/绿藻/其他）、清水、泥沙、阴影
2. 手动绘制藻类区域像素边界，确保类别划分准确、无漏标/错标
3. 保存标注文件：生成JSON格式标注文件 + 对应像素掩码图（Mask）
4. 输出：标注完成的数据集（含干净原图 + JSON标注 + 掩码图）
 
四、模块对接（唯一交付对象）
 
1. 对接对象：你 + 刘俊辉（AI视觉模型端）
2. 交付内容：
- 预处理干净图像集（统一格式、尺寸）
- LabelMe标注文件（JSON）+ 像素掩码图（Mask）
3. 对接要求：交付前完成数据归档，标注无错误、图像无损坏，确保模型端可直接读取训练
 
五、个人交付物（最终成果）
 
1. 完整的原始航拍图像归档
2. 预处理干净图像集（可直接用于标注）
3. LabelMe标注完成的全量数据集（含原图、JSON、掩码图）
4. 数据链路流程文档（标注规范、预处理参数记录）
